第一个坑是 没有标签却直接想训练分类器。这会卡死，因为论文核心不是单纯 CNN 分类，而是“带家族标签的监督式对比学习 + 分类”。

第二个坑是 敏感 API 列表不一致。如果你随手换一套 API 集合，后面图像、模型和结果都会漂。论文这 426 个 API 是固定来源，不建议自己随便改。

第三个坑是 multi-dex 没处理好。Androguard 的 Analysis 正是拿来把多个 DEX 合起来分析的，这一步别偷懒。

第四个坑是 调用图节点标识不统一。你后面要匹配 426 API，所以方法签名格式必须统一，不然会出现“图里明明有 API，但特征全是 0”的假象。

第五个坑是 把论文结果当成你自己 APK 集合也必须达到的数字。不行。只要数据集、标签质量、家族分布不同，F1 就会变；你真正应该对齐的是流程，不是盲对齐论文最终数值。
